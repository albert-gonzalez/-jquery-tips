Selectors

Jquery has a search engine which allows us to make queries with CSS selectors to retrieve DOM elements. The following tips show how to use them properly.

Avoid jQuery Selector Extensions

jQuery Selector Extensions cannot take advantage of the performance boost provided by the native querySelectorAll(). Try to use native css selectors if is possible.

Each

Iterate over the selected elements, executing a function for each matched element.

Inside the function, the keyword "this" refers to the DOM element, if you want to work with the jQuery object of this element execute $(this). This function receives two parameters, the index of the element and the element (equals to this).

Using each, avoid searching others DOM elements directly, try to find them relative to the current element (with functions like siblings, children, find, parent...). Use return false to break out of the loop early.

DOM Events

jQuery can listen to DOM events (click, change, submit...) on elements and add or change behaviour.

"on" or "shortcut"?

We have two ways to listen to events. The function jQueryObject.on() allow us to bind a handler to any kind of event. Most of events have a named method which is a shortcut to the on listener, like click, change, focus, etc. There is no difference between these two methods (but on allow you to add a namespace to the event), use the method which suits you.

Off and Namespaces

With the off method you can remove an attached event handler from an element. Namespaces are very useful to remove only the event handlers from a concrete event. The namespace of an event is defined in the on method, if you write "click.some-namespace" you are telling jQuery that this click event is of the "some-namespace" Namespace.

Preventing Default Action

In some cases, you may want to prevent the default action of some event, like submit a form or go to a link. All the event handlers are called with a parameter which has info and methods related to the event. One of these methods is preventDefault, which prevent the default action of the event. Returning false from an event handler will automatically call event.stopPropagation() and event.preventDefault().

Preventing Event from bubbling up

By default, a DOM event bubbles up from the element which triggers the event to its ancestors. Event.stopPropagation prevent the event from bubbling. Returning false from an event handler will automatically call event.stopPropagation() and event.preventDefault().

Keeping rest of the handlers from being executed

If an element has more than one event handler attached and you want that sometimes one of the handlers keeps the rest of the handlers from being executed you have to use Event.stopImmediatePropagation. This method prevents also the event from bubbling up. Note that handlers bound to an element are called in the same order that they were bound.
